## Content Governance

**Editorial Process**:
1. **Planning**: Monthly editorial meeting (editor + subject matter expert)
2. **Assignment**: Writer receives content brief + data sources
3. **Draft**: Writer submits to editor (Google Docs/Notion)
4. **Review**: Editor checks for accuracy (SME review required for technical posts)
5. **Approval**: CMO/Head of Content approves
6. **Publish**: Import to Next.js, add schema, internal links
7. **Promotion**: Social media, email newsletter, Medium repost

**Quality Standards**:
- All data claims must be cited (source, date accessed)
- All charts must be reproducible from GraphiQuestor data
- Statistical claims must include confidence intervals where applicable
- No forward-looking statements without "based on current data" disclaimer
- Peer review for technical content (de-dollarization methodology, sovereign stress formulas)

**SEO Checklist** (per post):
- [ ] Target keyword in title (first 60 chars)
- [ ] Target keyword in URL (hyphenated, lowercase)
- [ ] Meta title (60 chars, includes keyword)
- [ ] Meta description (155 chars, keyword + CTA)
- [ ] H1 matches title or close variation
- [ ] 3-5 H2 headings structured logically
- [ ] Internal links to 2-3 relevant pages
- [ ] External links to 1-2 authoritative sources
- [ ] Featured image with optimized filename + alt text
- [ ] 2-3 charts/graphics created in D3/Recharts
- [ ] Schema markup added (BlogPosting or appropriate)
- [ ] Last updated date added (if updating existing content)
