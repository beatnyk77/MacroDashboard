## Executive Summary

GraphiQuestor requires a **dual-pronged SEO strategy**:

1. **Authority Building** - Establish domain authority in the macro/sovereign risk space through thought leadership content
2. **Keyword Domination** - Target high-intent commercial keywords for specific macro niches (de-dollarization, sovereign stress, energy security, India/China macro)

**Baseline**: No existing organic search presence, minimal technical optimization  
**Projected Timeline**: 12-18 months to meaningful institutional traffic  
**Budget Tier**: High-touch B2B SaaS (akin to fintech/data platforms)

### Core Infrastructure

**Hosting & Performance**:
- Current: Vercel (good) - maintain
- Required: CDN with edge caching (Vercel has this)
- Target: All Core Web Vitals >90 (green)

**Core Web Vitals Targets**:
| Metric | Current | Target (3mo) | Target (12mo) |
|--------|---------|--------------|---------------|
| LCP | TBD | <2.5s | <2.0s |
| FID | TBD | <100ms | <50ms |
| CLS | TBD | <0.1 | <0.05 |

**Image Optimization**:
- All charts/graphics in WebP/AVIF with lazy loading
- SVG icons for UI elements
- Implement responsive images with `srcset`

**JavaScript Optimization**:
- Terminal UI uses heavy D3/Recharts - implement code splitting
- Only load data widgets on viewport intersection
- Defer non-critical JS

### Schema Markup Plan

**Per Page Type**:

- `/` (Home): `WebSite` + `PotentialAction` (SearchAction) + `Organization`
- `/pricing/`: `Product` with `OfferingCatalog`
- `/labs/[topic]/`: `ScholarlyArticle` or `Dataset` with `distribution`
- `/blog/[post]/`: `BlogPosting` with `author`, `datePublished`, `about` (macro topic)
- `/research/reports/`: `Report` with `datePublished`, `about`, `author`
- `/methodology/`: `FAQPage` + `HowTo` (data sourcing)
- `/case-studies/`: `CaseStudy` with `industry`, `knowsAbout`
- `/product/features/`: `SoftwareApplication` with `featureList`

**Markup Rivers**:
- Dataset pages: `Dataset` + `DataFeed` with `timeSeries`, `variableMeasured`, `measurementTechnique`
- Macro indicators: Use `FinancialDataset` schema where applicable
- API docs: `APIReference` + `TechArticle`

### AI Search Readiness

**Structured Data**:
- Implement schema.org markup on all content pages
- Use `answer` snippet markup for FAQ-style content
- Add `about`, `mentions` properties to link entities

**Content Optimization**:
- Each pillar page optimized for one primary keyword + 3-5 related
- Include definitions, calculations, examples (direct Q&A)
- "People also ask" sections: anticipate follow-up questions

**Entity Optimization**:
- Mention and connect: "de-dollarization", "BRICS", "sovereign debt", "net liquidity", "dedollar index"
- Use Wikipedia-style internal linking to connect concepts

### Mobile-First & Accessibility

- Terminal UI responsive for tablets (desktop-first but works on iPad Pro)
- All data visualizations accessible (ARIA labels, keyboard nav)
- Text contrast ratios meet WCAG AA
- Skip-to-content links for keyboard users
